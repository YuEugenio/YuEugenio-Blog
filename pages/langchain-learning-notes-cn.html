LangChain 学习笔记

围绕 LangChain 抽象、工作流编排与图结构 orchestration 的一组学习记录，并在页面下方直接展示对应源码的完整内容。

March 13, 2026

这一部分的重点不是“把模型调起来”本身，而是理解 LangChain 怎么把 prompt、模型、状态、工具和流程拆成可以复用、可以调试的组件。

1. 我主要在学什么

我把这一阶段当成“LLM 应用工程层”的入门：不是把所有逻辑都塞进一个大 prompt，而是学习怎么把理解问题、调用工具、获取结果、生成回答拆成清晰的步骤。

因此，LangChain 对我来说最重要的价值，是让多步任务的结构变得可见。这样当流程变复杂时，代码仍然可以读、可以调、可以继续扩展。

2. 我具体练了什么

  • 通过 `langgraphdemo.py` 理解状态图式工作流：把用户问题理解、真实搜索、答案生成拆成独立节点。
  • 通过 `langsmith_test.py` 观察一个最小 agent 例子如何绑定工具并完成一次调用。
  • 练习把模型配置和 API 信息改成环境变量，减少脚本对固定环境的耦合。

3. 我目前理解到的关键点

我觉得 LangChain 真正有价值的地方，在于任务一旦不再是“单轮问答”，它就能帮助我们把应用逻辑写得更像工程系统，而不是更像一次性 prompt 实验。

另外，我也越来越清楚：模型能力本身很重要，但流程设计同样重要。理解、检索、整合这三步如果拆得更清楚，结果往往也会更稳定。
