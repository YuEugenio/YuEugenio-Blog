OpenAI API 学习笔记

这一部分对我来说，是把“模型能力”真正变成“代码接口能力”的阶段。只有进入 API 层，很多抽象的能力才会变成可以被产品和脚本真正调用的对象。

1. 我主要在学什么

`openai_apis.ipynb` 是我这一部分的核心练习载体。我主要在理解：消息结构怎么写、不同调用方式差别在哪里、什么情况下需要流式输出或异步调用，以及多轮对话为什么必须显式维护历史。

2. 我具体练了什么

  • 基础 chat completion 调用。
  • 多模态输入的组织方式。
  • 异步请求的写法和并发调用场景。
  • 多轮对话的历史维护方式。
  • 流式输出以及“思考过程样式展示”的实验。

3. 我目前的理解

我现在更清楚地意识到，API 不是“把 prompt 发出去”这么简单。真正做应用时，消息历史、返回处理、异常处理、流式输出、模型参数配置，都会直接影响交互体验和工程可维护性。
