Agent 基础与 Demo

围绕工具调用、多步执行、流式输出与 ReAct 风格流程的一组早期 Agent 实践，并附完整源码。

March 13, 2026

这一部分是我从“普通聊天调用”往“能做事的系统”迈进的阶段。Agent 的关键并不只是回复像不像人，而是它能不能按步骤完成任务。

1. 我主要在学什么

我在这一阶段主要想搞清楚一个问题：当模型不再只是直接回答，而是需要调用工具、读取观察结果、再继续决策时，代码应该怎么组织。

2. 我具体练了什么

  • `draft.py`：练习流式回复和“思考样式内容”与正式回答内容的分离。
  • `travelassistant.py`：练习一个简化版 ReAct agent，能调用天气和搜索工具，按 Thought / Action / Observation 循环推进。

3. 我目前的理解

我现在比较清楚的是，Agent 的本质并不是“更聪明的对话”，而是“更结构化的执行”。一旦把动作格式、工具调用和观察结果都写清楚，系统就会比纯 prompt 更容易调试和演进。
